# ระบบยืม-คืนอุปกรณ์กีฬา แดชบอร์ดผู้ดูแลระบบ (Sports Lending System - Admin Dashboard)
โปรเจกต์ระบบบริหารจัดการคลังอุปกรณ์กีฬาและตรวจสอบสถานะการยืม-คืนแบบ Realtime สำหรับเจ้าหน้าที่ มหาวิทยาลัยกรุงเทพ

* **Link GitHub Repository:** https://github.com/irawansrasuphan/-bu-sport-Admin-
* **Link สำหรับทดสอบโปรแกรม (GitHub Pages):** https://irawansrasuphan.github.io/-bu-sport-Admin-/

---

## ฟังก์ชันการทำงานหลัก (Core Features)
1. **Realtime Stats Card:** แสดงจำนวนอุปกรณ์ทั้งหมดในคลัง (31 ชิ้น), กำลังถูกยืม (0 ชิ้น), และคงเหลือพร้อมใช้งาน โดยคำนวณและสอดคล้องกับฐานข้อมูลทันที
2. **Inventory Management:** ระบบปรับแต่งเพิ่ม-ลดจำนวนสต็อกอุปกรณ์กีฬาในคลังแบบ Realtime รองรับอุปกรณ์ 6 ประเภทหลัก (ลูกฟุตบอล, วอลเลย์บอล, บาสเกตบอล, ปิงปอง (แพ็ค), แบดมินตัน (ไม้), เทนนิส) เพื่อให้แอดมินควบคุมคลังได้ยืดหยุ่น
3. **Borrower Tracking:** แผงแสดงรายชื่อนักศึกษาที่ยืมเเละค้างส่งคืนอุปกรณ์ แยกรายละเอียดรายบุคคล เช่น ชื่อ-นามสกุล คณะ และเวลาที่ต้องส่งคืน เพื่อความสะดวกในการติดตามอุปกรณ์กีฬา

---

## เทคโนโลยีที่เลือกใช้ (System Prerequisites & Libraries)
ระบบพัฒนาขึ้นในรูปแบบ Web Application (Serverless Architecture) โดยอาจารย์ไม่จำเป็นต้องติดตั้งสภาพแวดล้อมหรือคอมไพเลอร์ใด ๆ บนเครื่อง เพียงมีเว็บเบราว์เซอร์และอินเทอร์เน็ตก็สามารถใช้งานได้ทันที ผ่านเทคโนโลยีดังนี้:
* **Frontend:** HTML5, CSS3 (Custom Responsive Layout) และ Vanilla JavaScript
* **Database:** Firebase Realtime Database สำหรับซิงค์ข้อมูลสแตทและจำนวนสต็อกอุปกรณ์แบบทันที

## ขั้นตอนการนำไปใช้และทดสอบโปรแกรม (How to Run & Test)

### วิธีที่ 1: ทดสอบผ่านระบบออนไลน์ (แนะนำสำหรับอาจารย์ผู้ตรวจ)
1. คลิกเข้าสู่ลิงก์บริการผ่าน GitHub Pages: `https://irawansrasuphan.github.io/-bu-sport-Admin-/`
2. ระบบจะโหลดข้อมูลสถานะจากฐานข้อมูล Firebase ขึ้นมาแสดงผลโดยอัตโนมัติ
3. สามารถทดสอบกดปุ่ม `+` หรือ `-` ในตารางปรับแต่งสต็อก เพื่อดูการเปลี่ยนแปลงของตัวเลขภาพรวมบนการ์ดสถิติแบบทันที

### วิธีที่ 2: รันโปรแกรมจาก Source Code บนเครื่องโลคอล (Local Setup)
1. ทำการ Clone หรือดาวน์โหลด Source Code จาก Repository นี้ลงบนเครื่องคอมพิวเตอร์
2. ตรวจสอบไฟล์ระบบภายในโฟลเดอร์หลัก ซึ่งต้องประกอบด้วยโครงสร้างไฟล์ดังนี้:
   ├── index.html   # หน้าโครงสร้างแดชบอร์ดหลัก
   ├── style.css    # ไฟล์ตกแต่งสไตล์ระบบ
   └── script.js    # ไฟล์ควบคุม Logic การคำนวณและเชื่อมต่อ Firebase
3. สามารถดับเบิ้ลคลิกเปิดไฟล์ `index.html` ผ่านเว็บเบราว์เซอร์ (เช่น Google Chrome, Microsoft Edge) เพื่อทดสอบการทำงานภายในเครื่องได้ทันที
